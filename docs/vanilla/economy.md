# Vanilla 4.3 — Economy Architecture

## Orbital Deposits (Space Resources)

### File Location
`common/deposits/01_orbital_deposits.txt` — primary orbital deposits (minerals, energy, research, strategic resources)

Also: `02_sr_deposits.txt` (strategic resources), `03_leviathans_deposits.txt`, `04_distant_stars_deposits.txt`, and DLC-specific deposit files.

### Structure

Each orbital deposit has:
- `resources` — category and production (`produces = { minerals = X }`)
- `station` — station class required (e.g., `shipclass_mining_station`)
- `is_for_colonizable` — whether it appears on habitable worlds
- `potential` — spawn conditions
- `habitat_modifier` — bonus districts when habitat is built over it
- `drop_weight` — spawn probability with conditional modifiers

### Economic Category

Orbital deposits use `orbital_mining_deposits` and `orbital_research_deposits` categories.

**Category hierarchy** (from `common/economic_categories/00_common_categories.txt`):
```
stations
  └─ station_gatherers          (line 594 — generates produces_mult, cost_mult)
  │    └─ orbital_mining_deposits   (line 887)
  │         └─ hab_mining_deposits  (line 895)
  └─ station_researchers        (line 604 — generates produces_mult)
       └─ orbital_research_deposits (line 891)
            └─ hab_research_deposits (line 899)
```

### Key Modifier for Scaling Space Resources

`station_gatherers_produces_mult` — multiplier on ALL mining station output. Generated by the `station_gatherers` economic category. Can be applied via technologies, traditions, edicts, static modifiers, etc.

Similarly: `station_researchers_produces_mult` for research stations.

These are the **cleanest levers** for globally scaling space resource production.

### Vanilla Orbital Yield Range

Minerals: `d_minerals_1` (1) through `d_minerals_5` (5)
Energy: `d_energy_1` (1) through `d_energy_5` (5)
Physics/Society/Engineering: similar 1-5 range
Strategic resources: typically 1-3 per deposit

### Scaling Approach

To make space resources outproduce planets:
1. **Buff deposit yields** — edit `01_orbital_deposits.txt` to increase base production
2. **Tech scaling** — create technologies that apply `station_gatherers_produces_mult`
3. **Era-based scaling** — use `on_yearly_pulse` events with game-year checks to upgrade deposits or apply modifiers
4. **New rich deposits** — create new deposit types (e.g., `d_minerals_10`) with higher yields and appropriate spawn weights

## Districts

### File Location
`common/districts/00_urban_districts.txt` and related files in `common/districts/`

### Structure

Each district has:
- `base_buildable` — whether it appears by default
- `min_for_deposits_on_planet` / `max_for_deposits_on_planet` — spawn range
- `resources` — upkeep and production
- `planet_modifier` — modifiers when built (housing, amenity, etc.)
- `triggered_planet_modifier` — conditional modifiers
- Job assignments (e.g., `job_miner_add = 2`)

### Key Levers for Nerfing Planetary Output

1. **Reduce jobs per district** — change `job_X_add` values
2. **Increase district upkeep** — modify `resources.upkeep` block
3. **Cap district slots** — `planet_max_districts` modifier, also `DEFAULT_MAX_DISTRICTS_PER_PLANET = 4` in defines (line 1599)
4. **Reduce job output** — modify individual jobs in `common/pop_jobs/`

## Planet Size

### Defines (`common/defines/00_defines.txt`)
- `DEFAULT_MAX_DISTRICTS_PER_PLANET = 4` (line 1599)

### How Planet Size Affects Districts
Planet size determines base district slots. Modifiable via:
- `planet_max_districts_add` — modifier to add/remove district slots
- Planet class definitions in `common/planet_classes/` control size ranges and spawn weights

### Capping Planet Size
To limit max planet size (target: 16-18) and shift distribution (more 12-14):
- Modify planet class definitions to change `planet_size` min/max ranges
- Adjust `spawn_odds` / size weights in planet class files
- Galaxy generation settings in `common/solar_system_initializers/` can influence system composition

## Economic Categories System

### File Location
`common/economic_categories/00_common_categories.txt`

### How It Works

Economic categories form a hierarchy. Each category can `generate_mult_modifiers` and `generate_add_modifiers` for `produces`, `cost`, and `upkeep`. This auto-generates modifiers like `category_name_produces_mult` that can be used anywhere modifiers are accepted.

### Key Categories for Our Mods

| Category | Line | Parent | Generated Modifier |
|----------|------|--------|-------------------|
| `station_gatherers` | 594 | `stations` | `station_gatherers_produces_mult` |
| `station_researchers` | 604 | `stations` | `station_researchers_produces_mult` |
| `orbital_mining_deposits` | 887 | `station_gatherers` | inherits parent modifiers |
| `orbital_research_deposits` | 891 | `station_researchers` | inherits parent modifiers |
| `planet_jobs` | — | `planets` | `planet_jobs_produces_mult` |
| `planet_deposits` | — | `planets` | `planet_deposits_produces_mult` |

Child categories inherit parent modifiers. Buffing `station_gatherers_produces_mult` affects all mining stations including habitats.

## Strategic Resources

### Deposit Files
- `common/deposits/02_sr_deposits.txt` — strategic resource orbital deposits
- Planetary strategic modifiers in `common/deposits/01_planetary_deposits.txt`

### Strategic Resource Types
Volatile Motes, Exotic Gases, Rare Crystals, Zro, Dark Matter, Living Metal, Nanites

### Approach: More Concentrated, Less Frequent
- Reduce `drop_weight` for strategic deposits across most systems
- Create new "rich" strategic deposits with high yields (e.g., 5-8 gases)
- Adjust `drop_weight` modifiers to concentrate them in fewer systems
- This makes specific systems strategically critical — worth fighting wars over

## Housing & Amenities

### Housing Deficit
When housing is insufficient, pops receive happiness and growth penalties. Severity scales with deficit size.

### Amenities Deficit
Amenity shortfalls reduce stability and happiness. Can be tuned via:
- `planet_amenities_no_happiness_add` — amenities deficit threshold modifiers
- Static modifiers for deficit penalties in `common/static_modifiers/`

### Design Goal: Increase Deficit Penalties
Modify the static modifiers that define what happens at various deficit levels to make them harsher. This punishes overloaded mega-planets and rewards spreading pops across multiple worlds.
