# Economy Overhaul — slice 2: bulk scaling parity (repeatable techs)
# =================================================================
# Design doc Track 1, levers 2-3: vanilla has infinite tile-output repeatables
# (planets scale forever at +5%/level) but NO station repeatable -> stations fall
# behind over the game. This file:
#   (A) OVERRIDES the 3 vanilla tile repeatables to a gentler +@econ_repeatable_per_level
#       /level (bloat control, Cetus-safe), and
#   (B) ADDS matching station-gatherers / station-researchers repeatables at the
#       SAME rate, so planet-job scaling and station scaling climb together and the
#       space>planet ratio set by slice 1 (structural cuts + station buffs) holds
#       as the game advances.
#
# OVERRIDE MECHANISM: (A) redefines vanilla tech KEYS. Technologies are keyed DB
# objects -> a same-key definition in a later-loaded file REPLACES the vanilla one.
# The "zzz_" filename sorts after vanilla's 00_*_tech_repeatable.txt so our versions
# win. These three keys are logged in docs/compatibility.md. Every field below is a
# faithful copy of vanilla 4.4.3 EXCEPT the modifier value (0.05 -> variable); if a
# future patch changes the other fields, re-sync this override.
#
# ⚠️ RUNTIME-VERIFY (file-inspection only): confirm in error.log that the three
# overrides replace (a benign "duplicate" log line may appear) and that the two new
# repeatables appear as research options after the finite chains complete. See README.

# =========================================================================
# (A) Vanilla tile-output repeatables — OVERRIDDEN to the gentler shared rate
# =========================================================================

	area = engineering
