# Economy Overhaul — lever #6: ESCALATING finite station-tech ramp
# =================================================================
# Design doc Track 1, lever #6 ("amplify the finite station techs"). Vanilla gives a
# FLAT +10% station output per tech across both finite chains, so only +50% by tier-3
# completion, then a dead zone until the (very-late) repeatables. We replace the flat
# +10% with an ESCALATING per-tier value so each later tech is worth more — giving space
# a real mid-game ramp that is reachable even at 2x tech cost (our likely MP setting).
#
#   MINING (engineering, station_gatherers): +10/20/30/40/50  -> +150% cumulative
#     buffs BOTH minerals AND energy — verified: energy orbital deposits ride
#     category=orbital_mining_deposits / shipclass_mining_station, identical to minerals,
#     so station_gatherers_produces_mult lifts both. (No separate energy tech exists or is
#     needed.) Nomad swap value escalated in parallel for consistency (nomads are MP-banned).
#   RESEARCH (physics, station_researchers): +10/15/20/25/30 -> +100% cumulative
#     gentler than mining: Track 2 keeps research PLANET-primary, space a supplement.
#
# OVERRIDE MECHANISM: technologies are keyed DB objects -> a same-key definition in a
# later-loaded file REPLACES the vanilla one (last-wins by load order; "zzz_" sorts after
# vanilla 00_eng_tech.txt / 00_phys_tech.txt). Every field below is a FAITHFUL copy of
# vanilla 4.4.3 EXCEPT the two modifier values (station mult + nomad swap mult), now driven
# by @econ_station_*_t* (econ_space_economy_variables.txt). The cost/weight @-vars
# (@tier1cost1 etc.) are vanilla inline tech variables — global once loaded, so referencing
# them here is valid (same pattern as zzz_econ_repeatable_techs.txt). These 10 keys are
# logged in docs/compatibility.md. RE-SYNC this file on any version bump that touches these
# techs (check error.log for a benign duplicate line confirming the override took).
