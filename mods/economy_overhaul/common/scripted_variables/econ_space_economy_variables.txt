# Economy Overhaul — "Space > Planets" tunable values
# ---------------------------------------------------
# All balance knobs for the space-vs-planet rebalance live here so the batched
# in-game test session can tune them in ONE place. These feed the country static
# modifier econ_space_primacy (see econ_space_economy_modifiers.txt), applied to
# every empire at game start (econ_on_actions.txt). Symmetric across all empires
# => MP-fair (no single player gains an advantage).
#
# Sign convention: buffs are positive, nerfs are negative. Values are mults
# (0.5 = +50%, -0.5 = -50%).

# --- Space strength: now carried by BUFFED DEPOSIT YIELDS, not a flat modifier ---
# v2 (2026-06-23): the old flat +50% station_*_produces_mult modifier was REMOVED to
# avoid base × flat-mult × repeatable compounding. "Space is primary" is now intrinsic
# to the deposits: common/deposits/01_orbital_deposits.txt is whole-file-replaced with
# PER-RESOURCE `produces` multipliers (v2.5/2026-06-26; tune them in that file's header):
# minerals ×1.40, energy ×1.60, research ×1.15; alloys/food/CG/trade left at vanilla.
# (Was a uniform ×1.75 in v2 → ×1.40 in v2.4 to fix the mineral/energy glut, open item #4.)
# Station repeatables (slice 2, @econ_repeatable_per_level below) still scale space over time.

# --- Housing scarcity (v2.1) — now done in the district files, not here ---
# Urban (city/hive/nexus) district housing is cut -30% directly in the whole-file
# replacement common/districts/00_urban_districts.txt (tune the 0.70 multiplier there).
# This replaced the old global @econ_housing_nerf / planet_housing_mult, which also hit
# rural housing and penalised wide play. Rural housing stays vanilla. Pairs with the
# tightened overcrowding defines (zzz_econ_defines.txt). #1 calibration target.

# --- Pop ASSEMBLY nerf (v2: bring assembly in line with logistic organic growth) ---
# Assembly is a FLAT rate (doesn't slow as a planet fills), organic growth is LOGISTIC
# (slows near capacity). v2.3: this nerf is now applied ONLY to NON-machine empires
# (via econ_organic_assembly_nerf, gated is_machine_empire = no in econ_overhaul.1).
# Why: machine empires grow SOLELY by assembly and are already heavily late-capped by the
# vanilla Country Growth Scale (an empire-pop-count tax on the flat channel that EXEMPTS
# logistic — confirmed in-game) — stacking our -33% on top guts them late, which we don't
# want. Organic empires keep it so robot ASSEMBLY can't be stacked tax-free on top of
# their organic LOGISTIC growth (the dual-channel abuse). Hives are inside the gate but
# have no assembly, so it's a no-op for them.
@econ_assembly_nerf = -0.33				# -33% pop assembly, NON-machine empires only (organics; machines exempt)

# --- Pop GROWTH nerf (v2.2: the spawning-pool / flat-growth parity gap) ---
# The assembly nerf above only touches planet_pop_assembly_* (machine/robot ASSEMBLY).
# Hive spawning drones, plantoid/lithoid BUDDING traits, and clone vats instead emit
# `bonus_pop_growth` — a FLAT additive growth that, like assembly, does NOT slow as a
# planet fills (unlike base logistic growth). Verified in-game (2026-06-24) that our
# assembly nerf missed this channel, so hives kept out-growing nerfed assembly empires.
# bonus_pop_growth_mult scales ONLY that flat-additive channel. It does NOT touch
# logistic_growth_mult, so base organic growth AND the Fertile / Rapid Breeders traits
# (both logistic_growth_mult) stay at vanilla as the reference curve. (The design doc's
# earlier worry that this hits Fertile was based on a wrong assumption — verified false.)
# Set equal to the assembly nerf so every FLAT-growth channel lands on one line; tune
# independently here if hive/budding/clone economies need a different slope than assembly.
@econ_growth_nerf = -0.33				# -33% flat additive pop growth (spawning pools, budding, clone vats), all empires

# --- Planetary primary-resource job output NERFS (OPTIONAL fine-tune — DEFAULT 0) ---
# SUPERSEDED as the primary planet-down lever. The structural slice
# (whole-file replacements: 00_scripted_variables.txt planet size cap 25->18 +
# 100_scripted_variables_zones.txt rural jobs 200->160) now does the planet-down work. Per the design doc (Track 1), this flat per-pop
# nerf becomes an OPTIONAL light fine-tune that must NOT stack hard on the
# structural cuts (double-nerf -> unemployment death spiral). Left at 0 (unused) —
# the modifier lines are removed from econ_space_primacy; re-wire them there only
# if playtest shows planets still over-produce AFTER the structural cuts.
# Per-resource variants are exactly what vanilla economy techs use at country scope
# (00_eng_tech.txt / 00_phys_tech.txt / 00_soc_tech.txt), so they cascade to all
# owned planets. Research deliberately never nerfed (vision targets PRIMARY resources).
@econ_planet_miners_nerf = 0			# fine-tune lever (unused); e.g. -0.2 = -20% planetary minerals
@econ_planet_technician_nerf = 0		# fine-tune lever (unused); e.g. -0.2 = -20% planetary energy
@econ_planet_farmers_nerf = 0			# fine-tune lever (unused); e.g. -0.2 = -20% planetary food

# --- Slice 2: bulk scaling PARITY (repeatable-tech per-level rate) ---
# ONE rate shared by all five repeatables (3 vanilla tile overrides + 2 new station
# techs) so planet-job scaling and station scaling "climb together but gently"
# (design doc Track 1, levers 2-3). Vanilla tile repeatables are 0.05/level and
# INFINITE (levels = -1) with NO station counterpart — that asymmetry is why space
# falls behind. We (a) NERF the tile rate 0.05 -> 0.03 (bloat control, Cetus-safe)
# and (b) ADD station gatherers/researchers repeatables at the SAME 0.03 so the
# space>planet RATIO set by slice 1 (structural cuts + station buffs) is preserved
# as the game advances rather than eroding. Lower this further if late-game numbers
# still bloat; raise if scaling feels too flat. See tech_repeatable_* in
# common/technology/zzz_econ_repeatable_techs.txt.
@econ_repeatable_per_level = 0.03		# +3% per repeatable level (vanilla tile was +5%)

# --- Slice 3: kilostructure build-limit reduction (multiplier taming, lever #12b) ---
# The Arc Furnace / Dyson Swarm build caps are `base 0 + sum of modifier:*_limit_add`
# (verified: common/script_values/04_script_values_machine_age.txt). So we lower the cap
# cleanly with a NEGATIVE country modifier (additive, no vanilla override) applied to
# every empire — fewer of these stacking compounders. Vanilla grants total ~5-6 per
# structure (megastructure tech + machine-age tech + an origin); -1 trims one slot.
# Empires without the unlocking tech have cap 0 and still can't build (harmless).
@econ_kilostructure_limit_reduction = -1	# -1 to each of arc-furnace / dyson-swarm caps

# --- Lever #6: ESCALATING finite station-tech ramp (the mid-game space ramp) ---
# Vanilla gives a FLAT +10% station output per tech across both finite chains
# (tech_space_mining_1..5 -> station_gatherers; tech_space_science_1..5 ->
# station_researchers), so +50% by tier-3 completion, then a dead zone until the
# repeatables (which gate behind the tier-3 chain end -> very late at 2x tech cost).
# We replace the flat +10% with an ESCALATING per-tier value so each later tech in the
# chain is worth more, giving space a real mid-game ramp that IS reachable at 2x cost.
# These override the vanilla techs (zzz_econ_finite_station_techs.txt) — every other
# field is copied verbatim; re-sync on a version bump. Logged in compatibility.md.
#
# MINING chain (engineering) buffs BOTH minerals AND energy — verified: energy orbital
# deposits ride category=orbital_mining_deposits / shipclass_mining_station, the SAME as
# minerals, so station_gatherers_produces_mult lifts both. No separate energy tech needed.
# Cumulative: 0.10+0.20+0.30+0.40+0.50 = +150% (vs vanilla +50%).
@econ_station_mining_t1 = 0.10
@econ_station_mining_t2 = 0.20
@econ_station_mining_t3 = 0.30
@econ_station_mining_t4 = 0.40
@econ_station_mining_t5 = 0.50
#
# RESEARCH chain (physics) buffs station_researchers. GENTLER escalation than mining:
# Track 2 keeps research PLANET-primary / space a supplement, so we lift the prize without
# flipping it to space-primary. Cumulative: 0.10+0.15+0.20+0.25+0.30 = +100% (vs vanilla +50%).
@econ_station_research_t1 = 0.10
@econ_station_research_t2 = 0.15
@econ_station_research_t3 = 0.20
@econ_station_research_t4 = 0.25
@econ_station_research_t5 = 0.30

# --- AI economic relief (planetary side) — applied to AI empires ONLY (is_ai = yes) ---
# The AI's economic planner is DEFICIT-driven: it reacts to homeless pops / resource
# shortages, not to "growth silently stalled." Our overcrowding nerf stops growth at
# 1.10x over-housing BEFORE any pop goes homeless, so the AI sees no signal and never
# builds residences -> it stalls at planet capacity and its economy never develops
# (confirmed in-game 2026-06-26). We can't make the AI understand the constraint, so we
# COMPENSATE: restore housing for AI empires only. Every real player is human (7-player
# MP), so the human-facing housing scarcity is untouched. Positive planet_housing_mult
# offsets the urban -30% cut (00_urban_districts.txt) + the tightened overcrowding
# (zzz_econ_defines.txt) for the AI. Applied via econ_overhaul.1, gated is_ai = yes.
# Generous on purpose ("AI is bad enough as-is"); tune down if AI over-sprawls.
@econ_ai_housing_relief = 0.30			# +30% planet housing, AI empires only
